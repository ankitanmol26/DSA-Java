1. what is an algorithm?
algorithm is a sequence of well designed steps used to solve a problem or perform a task.

2. what is a Time complexity?
Time complexity is a way to measure how the running time of an algorithm grows as the size of the input (n) increases. It describes the algorithm's efficiency rather than the exact time in seconds.

3. why don't we measure time in seconds?
because the time complexity in the dsa is measured how the algo grows as the size of the input increases. it is not measured in seconds or minutes.

4. why do we use Big O notation?
We use Big O notation because it provides a standard way to describe how an algorithm's running time or memory usage grows as the input size increases.

SPACE COMPLEXITY
1. What is Space Complexity?
ANS: amount of memory/space taken by the algorithm

2. What is Auxiliary Space?
ANS: the memory we create

3. Difference between Input Space and Auxiliary Space.
ANS: the memory which is already given to us is input space and the memory we create is auxiliary space

4. When is Space Complexity O(1)?
ANS: the extra memory is not created

5. When is it O(n)?
ANS: when the extra memory is created.

6.One example of O(n²) space?
ANS : matrix
